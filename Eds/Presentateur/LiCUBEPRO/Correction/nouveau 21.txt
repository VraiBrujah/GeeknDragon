Voici une version améliorée et clarifiée de ton prompt, structurée et sans ambiguïtés :
## Objectif

Créer un système centralisé de gestion de **toutes les variables** du projet (textes, chiffres, images), avec une **page d’administration visuelle et ergonomique** permettant de :

* Modifier ces variables de manière simple.
* Répercuter les changements automatiquement sur **toutes les pages concernées**.
* Faciliter la création de nouveaux produits en ne changeant que les variables, tout en gardant **la même structure HTML**.

## Règles de gestion des variables

1. **Textes identiques** → regroupés en **variables globales**.
2. **Textes uniques à une section** → stockés en **variables locales** spécifiques à la section.
3. **Noms des variables** → pertinents et explicites selon leur fonction ou position.

   * Exemple : `titrePrincipal`, `prixLocationProduitEDF`, `prixLocationConcurrent`.
4. **Variables éditables** → inclure **textes, chiffres, chemins d’images, labels, titres, descriptions**.

## Page d’administration (mode édition)

* **Interface moderne et intuitive** :

  * Vue arborescente ou tabulaire regroupant toutes les variables.
  * Champs éditables en direct (inputs, listes déroulantes, upload images).
  * Aperçu en temps réel (WYSIWYG) des pages avec les modifications.
* **Sauvegarde** :

  * Bouton “Appliquer” → les changements sont enregistrés dans les variables globales.
  * Impact immédiat sur toutes les pages qui utilisent ces variables.
* **Images** :

  * Tous les chemins d’images sont listés et éditables.
  * Dans le mode édition, on peut cliquer sur une image pour changer son fichier/son chemin.

## Version “édition des pages HTML”

* Une **copie complète** de toutes les pages HTML est placée dans un dossier d’édition.
* En mode édition :

  * Chaque champ texte est **cliquable et modifiable directement**.
  * Chaque image est **cliquable** → modification du chemin.
  * Les changements sont reflétés **en live dans l’aperçu**.
  * Lorsqu’on applique → sauvegarde dans les variables globales.
* La cohérence est garantie : toutes les pages partageant une même variable se mettent à jour automatiquement.

## Cas d’usage futur

* Permettre de **dupliquer un produit** en gardant la même structure HTML.
* Modifier uniquement les variables pour adapter le contenu (prix, titres, textes, images).
* Accélérer la création et la mise à jour de produits avec un **minimum d’effort manuel**.
